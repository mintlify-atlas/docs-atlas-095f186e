---

Aube is a fast Node.js package manager written in Rust. It works with your existing project without changing lockfile formats — if you have `pnpm-lock.yaml`, `package-lock.json`, `yarn.lock`, or `bun.lock`, aube reads and writes it in place. Warm CI installs run about 7x faster than pnpm and 3x faster than Bun.

## Why aube

<CardGroup cols={2}>
  <Card title="Fast installs" icon="bolt">
    Warm CI is ~7x faster than pnpm and ~3x faster than Bun. A global content-addressable store means package files are shared across all your projects.
  </Card>
  <Card title="Drop-in compatible" icon="plug">
    Works with your existing `pnpm-lock.yaml`, `package-lock.json`, `yarn.lock`, and `bun.lock`. No migration required to try it.
  </Card>
  <Card title="Secure defaults" icon="shield-check">
    New package releases wait out a minimum age before install. Lifecycle scripts from dependencies are blocked until you explicitly approve them.
  </Card>
  <Card title="Auto-install" icon="arrows-rotate">
    `aube run`, `aube test`, and `aube exec` check install freshness automatically. If something changed, they install first — no separate step needed.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Install aube">
    The recommended way is via [mise](https://mise.jdx.dev):

    ```bash
    mise use -g aube
    ```

    Or install from npm:

    ```bash
    npm install -g @endevco/aube
    ```
  </Step>
  <Step title="Run in your project">
    Drop aube into any existing Node.js project. If a lockfile is already present, aube reads and writes it in place:

    ```bash
    aube install
    ```
  </Step>
  <Step title="Use daily commands">
    ```bash
    aube add react          # add a dependency
    aube run build          # run a script (auto-installs if stale)
    aube test               # install + run tests
    aube exec vitest        # run a local binary
    ```
  </Step>
</Steps>
