aube is a Node.js package manager written in Rust. It drops into existing projects without requiring you to change your lockfile format — if your project already has `pnpm-lock.yaml`, `package-lock.json`, `yarn.lock`, or `bun.lock`, aube reads and writes it in place. The name means dawn in French and is pronounced `/ob/`, like "ohb".

## Why it's fast

aube is built in Rust and uses a content-addressable global store at `~/.local/share/aube/store/`. When you install the same package in multiple projects, the files are stored once and linked — not copied. On warm CI runs where the store is already populated, installs reach up to ~22x faster than pnpm and ~3x faster than Bun across fixture sets because aube only verifies and links instead of re-downloading.

The frozen fast path in aube's install pipeline short-circuits entirely when the lockfile and `package.json` match the recorded install state. Commands like `aube test` and `aube exec` check this state before running, so you avoid a redundant install on every invocation.

## Project status

<Note>
  aube is currently in beta. The current release is **v1.0.0-beta.12**. The CLI surface, lockfile format, and configuration are stable for day-to-day use, but some edges may change before v1.0.0. Breaking changes are noted in the release notes.
</Note>

aube is suitable for real projects today. Many teams are already using it in CI and local development. The beta label reflects that certain advanced features — such as patch management and some workspace edge cases — are still being refined.

<Tip>
  Because aube reads existing lockfiles, you can try it in a project without committing to it. Run `aube install` once, check the result, and switch back to your previous tool if needed — your lockfile stays in its original format.
</Tip>
